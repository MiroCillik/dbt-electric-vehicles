# dbt Electric Vehicle POC for Keboola

## Overview

A simple dbt project with a staging layer and three mart models, sourced from `out.c-dbt.Electric_Vehicle_Population_Data` in Keboola Storage. Initialized via `kbc dbt init` for Snowflake workspace setup and env var configuration.

## Source Data

- **Table**: `out.c-dbt.Electric_Vehicle_Population_Data`
- **Rows**: ~210K
- **Columns** (17): VIN, County, City, State, Postal Code, Model Year, Make, Model, Electric Vehicle Type, CAFV Eligibility, Electric Range, Base MSRP, Legislative District, DOL Vehicle ID, Vehicle Location, Electric Utility, 2020 Census Tract

## Models

### stg_electric_vehicles (staging)

Renames columns to snake_case, casts types (model_year as int, electric_range as int, base_msrp as int). Source: `{{ source('out.c-dbt', 'Electric_Vehicle_Population_Data') }}`.

### mart_by_make

Per manufacturer summary: vehicle count, avg electric range, BEV count, PHEV count, BEV percentage.

### mart_by_geography

Per county + city: vehicle count, top make (most registered), avg electric range.

### mart_cafv_eligibility

Per eligibility status + model year + EV type: vehicle count, avg range, avg MSRP.

## Technical Decisions

- All models materialized as `table`
- `kbc dbt init` handles workspace, profile, sources, and env vars
- Standard `ref()` chain: sources -> staging -> marts
- Basic schema tests (not_null, unique) on staging only

## Out of Scope

- Incremental models
- Documentation site
- CI/CD or deployment
- Advanced testing
